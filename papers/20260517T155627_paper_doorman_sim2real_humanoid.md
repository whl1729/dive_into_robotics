# 问题

人形机器人能后空翻、能打拳，为什么开不了门？

这不是玩笑。开门这件事把机器人最难的几个能力全绑在一起了：你得用一只晃动的头顶摄像头找到门把手，旋转一个弹簧加载的旋钮，跟踪门板画圆弧的轨迹，同时在铰链反力下保持全身平衡不摔倒。任何一环断了，整个动作就崩了。

之前的方案要么靠深度传感器、要么靠硬编码动作序列、要么靠人类遥操作——全都不可扩展。遥操作的人戴着VR头盔拿着手柄，连专家都经常判断不好门的弹力该给多大、门板弧线该怎么跟。

DoorMan 的回答是：完全在仿真里训练，只用 RGB 图像，零样本迁移到真实世界。而且比人类遥操作还快还准。

# 翻译

## 锚点：一个三阶段的「学徒出师」

把 DoorMan 的训练想象成一个学徒的成长过程：

第一阶段——**师傅开挂示范**。师傅（teacher policy）能看到学徒看不到的东西：门把手的精确3D坐标、手和门的接触力、机器人根部的速度。靠这些「作弊信息」，师傅用强化学习（PPO）学会了开门的全套动作。

第二阶段——**学徒看着学**。学徒（student policy）只有一双眼睛（RGB摄像头）和身体感觉（关节角度、角速度）。用 DAgger 方法，学徒在自己的视角下行动，师傅在旁边纠正：「你这步应该这样做。」学徒的视觉编码器和动作网络一起训练。

第三阶段——**学徒自己练**。学徒出师了，但因为只有眼睛没有透视眼，有些东西看不到（门把手被手挡住了、门板转过去了）。这时候用 GRPO（一种轻量 RL）让学徒自己摸索补偿策略——比如主动调整身体姿态让门把手保持在视野里。师傅从来没教过这招，是学徒自己「悟」出来的。

## 两个关键设计让这条路走通

**问题一：长序列探索的死循环。** 开门分六步（走近→预抓→抓握→旋转→推开→穿过），RL 训练时如果从头开始，机器人很难偶然完成后面的步骤，拿不到奖励就学不会，学不会就更拿不到奖励。

解法是「分段存档重开」（Staged Reset）。每当机器人第一次到达新阶段，存一个快照。下次重开训练时，不总是从头开始，而是按概率从中间某个存档点加载。这样后面的阶段也能被频繁练到。就像打游戏存档——你不用每次都从第一关打起才能练最终 Boss。

**问题二：仿真和真实的视觉鸿沟。** 如果只在一种门、一种光照下训练，到真实世界就傻了。

解法是在 IsaacLab 里搞「程序化生成」：5种门型、随机尺寸/重量/阻尼/弹力、5233种环境光照、PBR材质随机贴图。关键是——训练时见过的场景和测试时的真实场景没有任何重叠。不是复现真实世界，而是让策略见过足够多的变化，以至于真实世界只是「又一种没见过但不意外的变化」。

## 效果

在真实世界测试三类门（推杆开、拉杆开、推杠开），DoorMan 成功率 83%，专家遥操作 80%，新手遥操作 60%。完成时间上 DoorMan 平均 15.4 秒，比专家快 24%，比新手快 32%。

消融实验确认了每个组件的贡献：去掉视觉随机化，成功率从 85% 跌到 10-20%；去掉分段存档，teacher 训练直接卡死在第二阶段；去掉 GRPO 微调，student 成功率停在 50-70%，补不上和 teacher 的差距。

# 核心概念

## Staged Reset（分段存档重开）

一句话：训练时不总从头开始，而是从任务中间的存档点随机重开，让后期阶段也能被充分练到。

类比：想象你在学一首十分钟的钢琴曲。如果每次练习都从第一个音开始，你永远在磨前两分钟，后面的段落根本没机会练。分段存档就是让你有时候直接从第五分钟开始练，有时候从第八分钟开始练。每个段落都能得到足够的练习时间。

为什么重要：没有它，teacher 的 RL 训练在「抓门把手」这一步就卡死了——因为偶然成功抓到的概率太低，一旦抓的时候碰到门框还会被惩罚，策略反而学会了「别去抓」。分段存档打破了这个死循环。

## GRPO 微调（学徒自我进化）

一句话：用一种不需要 value function 的轻量 RL，让只有 RGB 输入的学生策略自己发现补偿「看不全」的技巧。

类比：你跟着教练学网球，教练能看到球的旋转方向（特权信息），你只能看到球飞过来的样子。教练教完基本动作后，你自己上场打，慢慢发现「如果我站位稍微偏左一点，就能更早看到球的轨迹」——这个站位技巧教练从来没教过，是你自己在受限视角下摸索出来的。

为什么重要：teacher-student 蒸馏后，student 的成功率只有 50-70%，因为 RGB 视角有遮挡和视野限制。GRPO 让 student 自己学会了「保持门把手在视野中心」「调整末端执行器姿态维持可见性」这些补偿行为，把成功率拉到 80-85%，逼近 teacher 的上限。

# 洞见

当感知受限时，最好的补偿策略不是来自更好的感知，而是来自行为本身的适应——学徒通过改变自己的站位和姿态，把「看不见」变成了「不需要看见」。

# 博导审稿

选题眼光很好。开门这个任务选得精准——它是 loco-manipulation 的试金石，把视觉、接触、平衡、长序列规划全绑在一起，而且有明确的成功/失败判据。不是人造的 benchmark，是真实世界的刚需。

方法上，三阶段 pipeline 不算新颖（teacher-student distillation 是老套路），但每个阶段的具体设计都有工程巧思。Staged Reset 解决长序列探索的思路简洁有效，GRPO 微调补偿 partial observability 的想法有启发性。不过整体更像是把已有技术精心组装，而非提出根本性的新方法。

实验诚意不错。和人类遥操作的对比是亮点——用同一个底层控制器，公平比较自主策略和人类操作者，这个 baseline 设置很有说服力。消融实验覆盖了主要组件。但 83% 的成功率意味着大约每 5 次有 1 次失败，论文对失败模式的分析偏少。另外，所有实验都在 Unitree G1 一个平台上做，泛化到其他机器人形态的能力未知。

写作上，Introduction 的开头那句斜体（「人形机器人的现实是，功夫和后空翻比开门先被解决了」）很抓人。整体结构清晰，但 Section 2.2 的数学推导（occupancy measure 那段）对于一篇系统论文来说有点过度形式化，实际贡献就是一个 reset 分布的混合，不需要那么多公式来说。

判决：**weak accept**。扎实的系统工程工作，在一个重要问题上给出了目前最好的端到端方案，但方法论层面的新贡献有限。

# 启发

**迁移**：Staged Reset 的思路可以直接用在任何长序列 RL 任务上。如果你在训练一个多步骤的自动驾驶决策策略（比如：变道→汇入→通过路口），也可以在每个阶段转换点存快照，让后期阶段得到充分训练。具体做法：在仿真器里检测状态机转换，缓存 100 个快照，reset 时按概率从各阶段采样。

**混搭**：GRPO 微调阶段的「学生自己发现补偿行为」这个范式，可以和 VLA（Vision-Language-Action）模型结合。先用 VLA 做 behavior cloning 得到基础策略，再用 GRPO 在仿真中微调，让模型学会在 partial observability 下的主动感知策略。这比纯粹加大训练数据可能更高效。

**反转**：这篇论文的隐含假设是「不需要深度信息，RGB 够用」。但它能做到这一点的前提是海量视觉随机化 + 50Hz 的高频控制。如果你的场景没有这么好的仿真器支持（比如非结构化户外环境），这个假设可能不成立——此时深度信息作为 invariant feature 的价值会回来。不要因为这篇论文就一刀切地抛弃深度传感器。
